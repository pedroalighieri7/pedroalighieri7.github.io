Quatro produtos, um único designer

( 1 ) Visão geral

Criando um sistema
do zero…o

Esse case study conta minha experiência de mais de 2 anos no founding team de uma startup. Um verdadeiro aprendizado o qual me deparei com o desafio de ser um time uma pessoa só, os erros e os acertos desenvolvidos no decorrer do projeto, criação de regras e normas, aplicação de teorias de UX a fim de desenvolver um sistema coeso com quatro produtos finais e de quebra uma identidade visual.

( 2 ) Papel

Território desconhecido

Depois de ter tido minha primeira experiência no setor público em uma empresa já consolidada. Recebi uma proposta para fazer parte do founding team de uma ideia. Tomei esse desafio como uma oportunidade de não só aprimorar minhas habilidades, mas colocar à prova tudo que eu aprendi em um ambiente de entregas rápidas com um time pequeno
Resultados
  • Experiência em varejo / Fintec: aprimoramento no setor de marketplace principalmente na área de cotação de preços, promoções e economia.
  • Aprimoramento de visualização de dados: criação de interfaces competentes com uma grande quantidade de informações visuais cruciais para o usuário.
  • Estudo em UX Research: aprimoramento das minhas habilidades de pesquisa na área e estudos com usuário a fim de aperfeiçoar telas, fluxos e decisões.
  • Quatro produtos em um: aprimoramento do sistema de gerenciamento da dados (back office), criação de dois aplicativos para o envio de dados e visualização de informações para sistema do cliente final (front office) e, por fim, desenvolvimento do sistema principal.
  • Designer com uma pitada de desenvolvedor: aprendi bastante com os meus companheiros de projeto. Através de uma perspectiva ágil, estivemos juntos na elaboração de features novas para o sistema, testes de usabilidade e uso de frameworks para otimização de tempo. Assim como aprimorei minhas habilidades em linguagens de programação básicas como HTML, CSS e JS.
  • Amadurecimento em gestão de projeto: organizar todas as etapas desde da prototipação incial em wireframes até os protótipos finais. Definindo demandas e gestão de tempo seguindo o esqueleto das metodolgias ágeis.

( 3 ) Principais desafios

O time de uma pessoa só

Desafio
  • Navegando sozinho: desenvolver, testar, analizar e aprimorar um sistema do zero sendo um único designer no começo da carreira.
  • Quatro vezes mais trabalho: desenvolver telas para quatro produtos com funções e formatos diferentes. Sendo elas: um banco de dados, dois aplicativos e o sistema principal.
  • Tempo vale muito: falta de espaço para testes iniciais de usabilidade devido a falta de tempo, orçamento inicial baixo e necessidade demonstrar valor profissional de forma rápida.
Solução
  • Sem inventar a roda: construção de alternativas de interface com base em bibliotecas gratuitas já estabelecidas como Material Design e Font Awesome.
  • A toque de caixa: protipação rápida de telas e features para testes de usabilidade, implementação e aprimoramento do sistema.
  • O mais importante primeiro: um sistema complexo deve ter o seu básico sólido. Meu foco foi tornar o fluxo mais importante do site o mais coeso e funcional possível. Assim, features que orbitavam esse sistema teriam uma fundação sólida na qual poderiam ser construídas e testadas com eficiência.
  • Valor invisível: sempre que possível destacar o valor do UX em elementos visuais. Dando mais espaço para pesquisa e coleta de dados quantitativos e qualitativos.

( 4 ) Desenvolvimento

Para batalha!
Ou quase isso...

Entregar primeiro,
detalhar depois Havia desde da concepção do projeto uma certeza: algumas coisas irião se tornar UX debit. Com o tempo correndo e vários devs de prontidão para implementação do novo sistema. Tive que escolher qual seria a prioridade e quais telas seriam refinadas posteriormente. Escolhi focar na funcionalidade principal do sistema, aproximadamente 40% do produto final, visto que as demais 60% dependiam dela em algum momento de seus fluxos.
Previsivelmente
necessário Boa parte do UX foi fundamentado na Jakob’s Law. Não havia tempo nem pessoal para desenvolver todos os elementos então foi necessário adotar funcionalidades que já tinham sido testadas e aprimoradas. Elementos e diretrizes do Excel, Kayak, Ifood e outras gigantes do mercado foram essenciais para concepção de assets, grids e comportamentos do sistema.
Breadcrump facilita o fluxo do usuário.
Estados de botões e seus comportamentos
Teste, pesquisa
e mais teste Após a finalização da primeira versão voltei minha atenção ao usuário. Felizmente a funcionalidade geral do sistema foi compreendida, faltando alguns ajustes que foram melhorados e depois testados novamente. O UX finalmente virou a grande estrela.
Complexilidade
facilitada Uma vez absorvido o feedback dos usuários no sistema, novas funcinalidade e elementos foram aprimorados com base em pesquisa, dados reais e dores externas ao empirísmo. Assim, o sistema consolidou-se com uma base concreta e eficaz. Um MVP com cara de produto final.

( 5 ) O fim de um capítulo

"Um crescimento gigantesco profissional. Muitas horas de pesquisa, muitos cafés e muitas lições que irei levarm, com certeza, na minha jornada como Product Designer"
Aprendizados
  • Finalizar sozinho: desenvolver quatro projetos foi uma responsibilidade enorme e poder finalizar foi algo maior ainda.
  • Crescimento como designer: resolução de problemas, administração de atividades e o poder da comunição foram elementos chave para finalizar esse projeto.
  • Um profissional sempre é um aprendiz: saber que não sabe e estar disposto a melhorar é a chave para o amadurecimento. Foi um ponto vital buscar mais informação, livros, artigos e relatos de outso designers nessa mesma situação.
  • Uma boa interface é resultado de um bom UX: o usuário é a chave para um sistema eficiente e real. Uma interface bonita tem sua função mas deve ter uma boa comunição com que irá utilizá-la.
  • Desenvolvimento rápido: ser capaz de desenvolver e progredir em um ambiente com mudanças constantes e geração de alternativas rápidas.
